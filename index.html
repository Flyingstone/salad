`USPS <http://statweb.stanford.edu/~tibs/ElemStatLearn/data.html>`_ Dataset.
Args:
    root (string): Root directory of dataset where ``processed/training.pt``
        and  ``processed/test.pt`` exist.
    train (bool, optional): If True, creates dataset from ``training.pt``,
        otherwise from ``test.pt``.
    download (bool, optional): If true, downloads the dataset from the internet and
        puts it in root directory. If dataset is already downloaded, it is not
        downloaded again.
    transform (callable, optional): A function/transform that  takes in an PIL image
        and returns a transformed version. E.g, ``transforms.RandomCrop``
    target_transform (callable, optional): A function/transform that takes in the
        target and transforms it.

salad.layers.vat.ConditionalEntropy

estimates the conditional cross entropy of the input

$$
rac{1}{n} \sum_i \sum_c p(y_i = c | x_i) \log p(y_i = c | x_i)
$$

By default, will assume that samples are across the first and class probabilities
across the second dimension.

salad.solver

Model optimization by stochastic gradient descent and variants The general structure:

  • Each experiment configuration is a subclass of Solver or some derivative loss functions
  • Solvers only specify how data and models are used to generate the losses
  • Similarities between deep learning experiments (checkpointing, logging, ...) are implemented in the Solver class.

In general, for many experiments, it makes sense to set up a solver as a subclass of a specific other solver; i.e. when the general problem is concerned with classifcation, a CrossEntropySolver would be a natural choice.

Classes where designed with the possibility of re-use in mind. The goal is to exploit the particular structure most deep learning experiments share.

salad.solver.base

salad.solver.base.EventBasedSolver
salad.solver.base.Solver

General gradient descent solver for deep learning experiments

Makes very little assumptions about the overall structure of the training
routine, except

- one or multiple models are used
- the dataset can be fed to an instance of the torch.util.data.DataLoader

To set up your own experiment, follow these steps:

- Specify your optimizer(s)
- Specify how losses are derived given your models and a batch of data

Assumption:

- multiple models, multiple losses, multiple optimizer

- per model:
    - multiple optimizers for different parts of the model
    - per optimizer:
        - multiple losses

GAN:
    - Discriminator
        optim D
            losses
    - Generator
        optim G

--> loop over models, then loop over optimizers, then aggregate losses
